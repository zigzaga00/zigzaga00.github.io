    In every secure system, not all users are equal. Some have clearance to view confidential data, others can only access what they need. To succeed in your next mission, you‚Äôll need to understand how Linux handles users, groups, and privileges.

    üë§ Types of Linux Users

    Linux systems have different user roles:

    • root ‚Äì üëë The one true superuser. Has complete control. There is only one root user per system.
    • Regular users ‚Äì Like you. Access is limited to keep the system safe.
    • System users ‚Äì Used by services and background processes (e.g., daemon, syslog).

    To see who you are:

    whoami

    To see all users:

    cat /etc/passwd

    üß™ sudo ‚Äì Power on Demand

    Some trusted users are allowed to temporarily act like root using the sudo command:

    sudo <command>

    This is only possible if you're in a special group ‚Äì usually called sudo or wheel.

    Check if you're in the sudo group:

    groups

    Note

    The original meaning of sudo was super user do as when using sudo a regular user can do as the super user (root) can do

    üï∏Ô∏è Linux Groups: The Network of Trust

    Every user can belong to one or more groups. Groups help control who can:

    • Read sensitive logs
    • Access protected directories
    • Run powerful commands

    Example groups:

    • adm ‚Äì Can read log files
    • sudo or wheel ‚Äì Can use the sudo command
    • www-data ‚Äì Used by web servers

    See which groups you belong to:

    groups

    Check another user:

    groups otherUser

    üóÇÔ∏è /etc/passwd and /etc/group

    Two files tell you everything about users and groups:

    • /etc/passwd ‚Äì Lists all user accounts
    • /etc/group ‚Äì Lists all groups and who is in them

    Note

    Passwords are NOT stored in /etc/passwd - they are found in /etc/shadow but this is a restricted file and even then the passwords of users have been hashed so they cannot easily be read

    Explore with:

    cat /etc/passwd
cat /etc/group

    üîç Want to find all users in the adm group?

    grep adm /etc/group
